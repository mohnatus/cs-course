# Сложение

Сумматор в компьютере умеет складывать двоичные числа (учитывая перенос разрядов). Но не умеет вычитать. Поэтому вычитание реализуется через сложение.

## Пример на десятичных числах: 

```
273 - 191 = 
```

### Решение 1 (дополнение до 9):

```
273 - 191 = 
273 - 191 + 999 - 999 =
273 + (999 - 191) - 999 = // это дополнение до 9
273 + 808 - 999 =         // сумму можем посчитать
1081 - 999 = 
```

Тут снова нужно вычитать, это плохо.

### Решение 2 (дополнение до 10):

```
273 - 191 = 
273 - 191 + 1000 - 1000 =
273 + (1000 - 191) - 1000 =  // это дополнение до 10
```

Дополнение до 10 сложно считать, но его можно представить в виде Дополнение до 9 + 1:

```
273 + (999 - 191 + 1) - 1000 = 
273 + (999 - 191) + 1 - 1000 = 
273 + 808 + 1 - 1000 = 
1082 - 1000 = 
1_082 - 1_000 =       // это легко вычесть
82
```


## Пример на двоичных числах: 

### 1: Результат положительный

```
1010 - 0100 =                        // 10 - 4
1010 - 0100 + 10000 - 10000 =  
1010 + (10000 - 0100) - 10000 =      // дополнение до 2
1010 + (1111 - 0100 + 1) - 10000 = 
1010 + (1111 - 0100) + 1 - 10000 =   // дополнение до 1
1010 + ~0100 + 1 - 10000 =           // заменяем на инверсию
1010 + 1011 + 1 - 10000 = 
1010 + 1100 - 10000 = 
1_0110 - 1_0000 =                    // переполнение
0110                                 // 10 - 4 = 6                    
```

### 2: Результат отрицательный

```
0100 - 1010 =                        // 4 - 10
0100 - 1010 + 10000 - 10000 = 
0100 + (9999 - 1010) + 1 - 10000 = 
0100 + ~1010 + 1 - 10000 = 
0100 + 0101 + 1 - 10000 = 
1001 + 1 - 10000 = 
1010 - 1_0000 =                       // переполнение НЕ произошло
```

тут снова нужно сделать дополнение до единицы:

```
1010 - 1_0000 = 
1010 - (1111 + 0001) = 1010 - 1111 - 0001 =
- (1111 - 1010 + 0001) = 
- (~1010 + 0001) = 
- (0101 + 0001) = 
- (0110) = 
-6
```
