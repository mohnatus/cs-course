## Пример сложения двух BCD-чисел

```
678 = 0110_0111_1000
 67 = 0000_0110_0111
```

Сложение по разрядам:

```
   0110_0111_1000
+  0000_0110_0111
   --------------
            _1111
```

Это невалидное BCD-число, поэтому к нему нужно прибавить 6 (0110). Так как это разница между возможными 16 комбинациями и 10, которые используются (6 комбинаций не используются).

```
   0110_0111_1000
+  0000_0110_0111
   --------------
            ^^^
            _1111
             0110
             ----
            10101
```

Это переполнение пойдет в следующий разряд:

```
   0110_0111_1000
+  0000_0110_0111
   --------------
        ^^^^_0101
        0111 
        0110
        ----
        1110   
```

Это снова невалидное BCD-число, к нему нужно прибавить 6, чтобы вернуть в стандартную форму:

```
   0110_0111_1000
+  0000_0110_0111
   --------------
       ^^^
        1110_0101
        0110
        ----
       10100   
```

Переполнение уходит в следующий разряд:

```
   0110_0111_1000
+  0000_0110_0111
   --------------
      ^
   0110_0100_0101 
   0000
   ----
   0111
```

Итого:

```
   0110_0111_1000  // 678
+  0000_0110_0111  // 067
   --------------
   0111_0100_0101  // 745
```

## Вычитание BCD-чисел

Используем дополнение до 9:

```
678 - 067 = 
678 + (999 - 067) = 
678 + 932 = 
```

Переводим в BCD:

```
  0110_0111_1000
+ 1001_0011_0010
----------------
            1000
            0010
            ----
            ^^
            1010  // невалидный BCD
            0110
            ----
           x0000  // переполнение
       ^^^^
       0111       // невалидный BCD
       0011
       ----
      ^^^
       1011       // невалидный BCD
       0110
       ----
      x0001        // переполнение   
 ^^^^^
  0110
  1001
  ----
 10000             // невалидный BCD
  0110
  ----
 x0110             // переполнение
 ---------------
 x0110_0001_0000   
```

Переполнение возвращается в крайний правый разряд:

```
x0110_0001_0000 =>
 0110_0001_0001    // 611 = 678 - 067
```

## Вычитание с отрицательным результатом

```
067 - 678 = 
067 + (999 - 678) = 
067 + 321 =
```

Переводим в BCD:

```
  0000_0110_0111
  0011_0010_0001
  --------------
  0011_1000_1000 // 388
  --------------
```

Переполнения не случилось. Результат получился положительный, а должен быть отрицательный. 

Запоминаем, что он отрицательный (знак нужно хранить в отдельном - первом -  полубите).

Чтобы получить правильный, нужно сделать дополнение до 9:

```
     0011_1000_1000  // 388

     999 - 388 = 611

     0110_0001_0001 // 611

1000_0110_0001_0001 // -611
```
